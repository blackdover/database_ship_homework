问题与解决（最简形式）

- 问题：`Task.From_Slot_ID` 与 `Task.To_Slot_ID` 在建表中声明为 NOT NULL，但样例数据有 NULL 值，导致插入失败（违反 NOT NULL）。  
  解决：将两列改为可空（INT NULL），并在样例数据中尽量填入合理的箱位编号以保持数据语义。 修改文件：`sqlutil/init_database.sql`, `sqlutil/init_tables_only.sql`, `sqlutil/init_sample_data.sql`。

- 问题：`Party.Party_Name` 有唯一约束（UQ_Party_Name），插入重复名会报错。  
  解决：对插入脚本采用幂等写法（ON DUPLICATE KEY UPDATE ...）或在插入前检查是否存在。 修改文件：`sqlutil/init_sample_data.sql`（已改为 ON DUPLICATE KEY UPDATE）。

- 问题：表中存在传递依赖（例如 Party: City -> Country），不严格满足 3NF（存在冗余）。  
  解决建议：可保留冗余以换取性能，但需在数据维护流程或触发器中保证一致性；或拆出地理字典表以完全规范化（带来额外 JOIN 成本）。

- 问题：部分唯一/外键约束（如 Container_Number, Port_Code, Username/Email）会导致导入脚本重复运行时失败。  
  解决：使用幂等插入（ON DUPLICATE KEY UPDATE）或先删除/清空目标表（慎用）。

- 其他建议：对业务约束（Task 类型与箱位必填关系）可通过触发器或 CHECK 进行业务规则校验（可选）。
